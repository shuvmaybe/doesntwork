Stop using 'Don't work' or 'Doesn't work'

Save your time and everyone else's by being clear and specific.

It's incredibly frustrating to see messages like "I followed all the steps, but it doesn't work" without any further details. How are we supposed to help you with that?

When you encounter an issue, provide the necessary details right away. Here's what you should include:

  • What is not happening: Clearly state what isn't happening as expected. For example: "I installed this theme, but the background is white instead of black."
  • What should happen: Describe the expected outcome. For instance: "The theme should display a black background."
  • Provide error messages & additional information: Include any error messages and relevant context. For example: "No error messages, but the background color is incorrect."

Here's an example of how using "Doesn't work" wastes time:

[10:15 AM] [Joe Biden] Hello everyone, I installed this theme and it doesn't work.
[10:17 AM] [Donald Trump] What specifically isn't working?
[10:18 AM] [Joe Biden] Do I need a different plugin for the theme?
[10:19 AM] [Donald Trump] What is the issue with the theme?
[10:21 AM] [Joe Biden] It's all white, but it's supposed to have a black background.
[10:22 AM] [Donald Trump] The background link for themes has expired. Replace your background link with this one. [solution]
[10:23 AM] [Joe Biden] Works! Thanks!

It took Joe 8 minutes to get his problem fixed, just because he started with "doesn't work" instead of explaining the problem right away.

What Joe should have done was explain the issue he was encountering and what he expected to happen. In this case, his theme was supposed to have a black background, but it was white.

[10:15 AM] [Joe Biden] Hello everyone, I was installing this theme and it is completely white instead of having a dark background. How can I fix this?
[10:17 AM] [Donald Trump] The background link for themes has expired. Replace your background link with this one. [solution]
[10:18 AM] [Joe Biden] Works! Thanks!

Joe's issue was now fixed in 3 minutes! 62.5% Faster than with just saying 'doesn't work' every couple of messages.

When 'Doesn't work' is acceptable:

While the importance of explaining your issue right away is emphasized and resolved, there are times when using "doesn't work" is appropriate:

For example, when seeking solutions known to have similar error messages or identical situations, such as:

[10:30 AM] [Barack Obama] Hello, my code seems to error due to *error 1*, my code is: [code]
[10:32 AM] [Donald Trump] It seems like you are using the function improperly. [solution]
[10:33 AM] [Barack Obamna] Doesn't work.
[10:34 AM] [Donald Trump] Ah, I see the issue now. [solution]
[10:34 AM] [Barack Obama] Now it works! Thanks.

Here, Obama used "Doesn't work" effectively to quickly resolve his issue, as Donald was already aware of the context from earlier messages.

If Obama wanted to let Donald know there is a different issue, he could still use 'Doesn't work', but also provide the new issue he is encountering.

[10:30 AM] [Barack Obama] Hello, my code seems to error due to *error 1*, my code is: [code]
[10:32 AM] [Donald Trump] It seems like you are using the function improperly. [solution]
[10:33 AM] [Barack Obama] Doesn't work. *error 2*
[10:34 AM] [Donald Trump] Ah, I see the issue now. [solution]
[10:34 AM] [Barack Obama] Now it works! Thanks.

Remember, providing clear details upfront not only helps you get faster solutions but also respects everyone's time. Be more like Obama.
